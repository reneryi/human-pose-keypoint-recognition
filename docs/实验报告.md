## 五、任务一：版本管理与 Git 基础认识

版本管理可以记录项目从初始化到每次修改的完整历史。与简单文件备份相比，Git 不仅保存文件内容，还保存提交说明、提交时间、作者信息、分支关系和版本标签。因此，Git 更适合软件开发中的需求迭代、缺陷修复和版本发布。

本项目中使用 `master` 分支作为稳定主分支，使用 `feature/video-pose` 分支开发视频识别功能，并通过 `v1.0.0` 和 `v1.0.1` 两个标签标记正式版本。

## 七、v1.0.0 图片人体关键点识别实现

图片版本使用 MediaPipe Pose 识别人体关键点，使用 OpenCV 绘制人体骨架，使用 Streamlit 提供本地 Web 界面。

### 1. 输入

用户在页面中上传一张图片，支持 `jpg`、`jpeg`、`png` 和 `bmp` 格式。

### 2. 处理流程

1. Streamlit 读取用户上传的图片。
2. PIL 图片转换为 OpenCV BGR 格式。
3. BGR 图片转换为 RGB 格式，输入 MediaPipe Pose。
4. 如果检测到人体姿态，则绘制关键点和骨架连接线。
5. 将结果图片保存到 `outputs/image_result/`。

### 3. 输出

程序在页面中展示原图与结果图，并提供结果图片下载按钮。

### 4. 测试情况

已使用 `python -m py_compile app.py pose_image.py` 完成语法检查，并调用图片识别函数生成测试输出图片，说明 v1.0.0 的核心功能可以运行。

## 八、v1.0.1 视频人体关键点识别实现

视频版本在 `feature/video-pose` 分支中开发，完成后合并回 `master` 分支并发布 `v1.0.1` 标签。

### 1. 输入

用户在页面中上传视频文件，支持 `mp4`、`avi`、`mov` 和 `mkv` 格式。

### 2. 处理流程

1. Streamlit 将上传视频保存为临时文件。
2. OpenCV 使用 `VideoCapture` 逐帧读取视频。
3. MediaPipe Pose 对每一帧进行人体关键点识别。
4. OpenCV 将关键点和骨架绘制到视频帧上。
5. 使用 `VideoWriter` 将处理后的视频帧导出为 `.mp4` 文件。

### 3. 输出

程序输出处理后视频，保存到 `outputs/video_result/`，并在页面中展示处理后视频和下载按钮。

### 4. 测试情况

已使用临时生成的短视频进行测试，处理结果如下：

```text
processed=6, pose_frames=0, size=5902
```

测试视频不包含真实人体，因此 `pose_frames=0` 是合理结果；结果视频文件存在且大小大于 0，说明视频处理和导出流程通过测试。
