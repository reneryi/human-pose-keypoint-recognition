# 实验报告

## 一、实验名称

基于 Git 与 GitHub 的人体关键点识别软件版本管理实验

## 二、实验目的

1. 理解软件版本管理在开发、协作、问题回溯和版本发布中的作用。
2. 掌握 Git 的工作区、暂存区、本地仓库和远程仓库等基本概念。
3. 掌握 `git status`、`git add`、`git commit`、`git log`、`git diff`、`git branch`、`git switch`、`git merge`、`git pull`、`git push` 和 `git tag` 等常用命令。
4. 通过人体关键点识别软件原型，完成从需求实现、测试验证到版本发布的完整流程。

## 四、实验内容概述

本实验以人体关键点识别软件为载体，完成两个版本的开发：

1. `v1.0.0`：图片人体关键点识别。
2. `v1.0.1`：视频人体关键点识别。

同时在开发过程中记录 Git 操作，体现本地仓库、远程仓库、分支开发、合并和标签发布。

## 五、任务一：版本管理与 Git 基础认识

版本管理可以记录项目从初始化到每次修改的完整历史。与简单文件备份相比，Git 不仅保存文件内容，还保存提交说明、提交时间、作者信息、分支关系和版本标签。因此，Git 更适合软件开发中的需求迭代、缺陷修复和版本发布。

本项目中使用 `master` 分支作为稳定主分支，使用 `feature/video-pose` 分支开发视频识别功能，并通过 `v1.0.0` 和 `v1.0.1` 两个标签标记正式版本。

## 六、任务二：本地仓库与文件基本操作

本项目从旧仓库继续操作，但旧文件无参考价值，因此先将旧项目文件清空并提交独立记录。随后重新创建项目结构、说明文档、依赖文件和输出目录，从而体现文件新增、修改、删除、暂存、提交、查看差异和查看历史等基本 Git 操作。
